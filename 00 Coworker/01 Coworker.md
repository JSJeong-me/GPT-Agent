OpenCoworker는 단순 챗봇이 아니라, 사용자가 허용한 폴더와 도구를 사용해 **문서·스프레드시트·보고서 같은 결과물**을 만드는 데스크톱 AI Agent입니다. 공식 문서에서도 파일 읽기, 메시지/메일 작업, PDF·문서·스프레드시트 생성, 예약 자동화, 로컬 보관과 승인 기반 실행을 핵심 기능으로 설명합니다. ([GitHub][1]) aisuite 관점에서는 Python 함수, toolkits, MCP 서버, 정책 기반 실행 제어를 붙여 Agent를 구성하는 구조로 이해하면 좋습니다. ([GitHub][2])

# 초급자 실습 시나리오: Desktop AI Coworker 이해하기

## 1. 실습 제목

**“AI에게 일을 맡기기: Desktop AI Coworker로 파일 정리, 요약, 표 작성, 승인 과정을 체험하기”**

## 2. 실습 대상

* 생성형 AI를 사용해 본 경험은 있지만 Agent 개념은 처음인 학습자
* 코딩 경험이 거의 없는 일반 사무직, 기획자, 교육생
* ChatGPT와 같은 챗봇과 Desktop AI Coworker의 차이를 이해하고 싶은 초급자

## 3. 실습 목표

실습이 끝나면 학습자는 다음을 설명할 수 있어야 한다.

1. Chatbot과 Desktop AI Coworker의 차이
2. AI Agent가 파일을 읽고, 계획을 세우고, 결과물을 만드는 과정
3. 폴더 접근 권한, 읽기 전용, 쓰기 권한, 승인 절차의 의미
4. 사람이 반드시 검토해야 하는 지점
5. Desktop AI Coworker를 실제 업무에 적용할 수 있는 간단한 시나리오

## 4. 핵심 개념

Desktop AI Coworker는 “질문에 답하는 AI”가 아니라 “업무를 맡길 수 있는 AI 동료”이다.

일반 챗봇은 주로 대화창 안에서 답변을 생성한다. 반면 Desktop AI Coworker는 사용자가 허용한 범위 안에서 로컬 파일을 읽고, 내용을 정리하고, 보고서나 스프레드시트 같은 결과물을 만들 수 있다. 따라서 중요한 개념은 다음 네 가지이다.

* **Context**: AI가 참고할 수 있는 파일, 폴더, 문서
* **Tool**: 파일 읽기, 문서 작성, 표 생성, 검색, 일정 실행 같은 도구
* **Permission**: 사용자가 허용한 범위 안에서만 작업하는 권한
* **Human Approval**: 위험하거나 중요한 작업은 사람이 검토하고 승인하는 절차

# 단계별 실습 시나리오

## Step 1. Chatbot과 Desktop AI Coworker 비교

### 강사 설명

먼저 일반 챗봇에게 다음과 같이 질문한다.

```text
고객 문의 5건을 중요도별로 분류하고 대응 방안을 정리해 주세요.
```

학습자는 챗봇이 실제 파일을 보지 못하면 정확한 답을 할 수 없다는 점을 확인한다.

이후 Desktop AI Coworker에는 실습 폴더를 연결하고 같은 요청을 한다. 이 차이를 통해 “대화형 AI”와 “업무 수행형 AI”의 차이를 이해한다.

### 학습 포인트

* 챗봇은 사용자가 입력한 내용 안에서 답변한다.
* Desktop AI Coworker는 허용된 파일을 읽고 업무 맥락을 파악한다.
* AI가 직접 접근할 수 있는 Context가 결과 품질을 좌우한다.

---

## Step 2. 실습 폴더 접근 권한 설정

### 학습자 작업

OpenCoworker 또는 유사한 Desktop AI Coworker 도구에서 다음 폴더를 연결한다.

```text
desktop-ai-coworker-practice/
```

처음에는 **읽기 전용 권한**으로 시작한다.

### 강사 설명

읽기 전용 권한은 AI가 파일 내용을 참고할 수는 있지만 기존 파일을 변경하지 못하는 상태이다. 초급자 실습에서는 먼저 읽기 전용으로 시작하고, 결과물을 저장할 때만 쓰기 권한을 허용하는 방식이 안전하다.

### 학습 포인트

* AI에게 모든 폴더를 열어 주지 않는다.
* 실습용 폴더만 연결한다.
* 실제 개인정보, 계약서, 민감한 회사 자료는 사용하지 않는다.
* 권한 부여는 “AI가 할 수 있는 행동의 경계”를 정하는 일이다.

## Step 3. AI에게 업무 파악을 먼저 시키기

### 프롬프트

```text
이 실습 폴더의 구조를 파악해 주세요.
각 폴더에 어떤 종류의 파일이 있는지 설명하고,
이 파일들을 바탕으로 어떤 업무를 수행할 수 있을지 5가지 아이디어를 제안해 주세요.

아직 파일을 수정하거나 새 파일을 만들지는 마세요.
먼저 계획만 보여 주세요.
```

### 기대 결과

AI는 다음과 같이 응답해야 한다.

* 고객 문의 파일이 들어 있는 폴더
* 영수증 데이터가 들어 있는 CSV 파일
* 제품 FAQ와 서비스 정책 문서
* 가능한 업무 예시

  * 고객 문의 분류
  * 긴급도 판단
  * 대응 우선순위 표 작성
  * 월간 비용 리포트 작성
  * FAQ 기반 고객 응답 초안 작성

### 학습 포인트

Desktop AI Coworker에게 바로 “작업 실행”을 시키기 전에 먼저 “계획”을 요청하는 것이 좋다.

* Desktop AI Coworker의 핵심은 자동화가 아니라 “통제 가능한 자동화”이다.

# 강사용 진행 가이드

## 실습 전 설명할 핵심 문장

“Desktop AI Coworker는 여러분의 컴퓨터에서 일하는 AI 동료입니다. 하지만 모든 권한을 가진 동료가 아니라, 여러분이 허용한 폴더와 도구 안에서만 일하는 제한된 동료입니다.”

## 실습 중 강조할 점

* AI에게 업무를 맡길 때는 목적, 입력 자료, 출력 형식, 제한 조건을 명확히 말해야 한다.
* AI가 만든 결과물은 최종본이 아니라 검토가 필요한 초안이다.
* 파일 삭제, 메일 발송, 외부 공유, 결제, 계약 관련 작업은 사람의 승인이 필요하다.
* 민감한 파일은 실습에 사용하지 않는다.

## 초급자용 프롬프트 공식

```text
[역할] 당신은 나의 업무 보조자입니다.
[입력] 다음 폴더/파일을 참고하세요.
[작업] 무엇을 해 주세요.
[출력] 어떤 형식으로 만들어 주세요.
[제한] 무엇은 하지 마세요.
[승인] 실행 전에 계획을 먼저 보여 주세요.
```

## 예시

```text
당신은 나의 업무 보조자입니다.
01_customer_inquiries 폴더의 고객 문의 파일을 참고하세요.
문의를 유형별, 긴급도별로 분류해 주세요.
결과는 표 형식의 markdown 보고서로 작성해 주세요.
원본 파일은 수정하지 마세요.
파일을 생성하기 전에 먼저 계획을 보여 주세요.
```

# 평가 루브릭

| 평가 항목      | 우수                         | 보통          | 보완 필요              |
| ---------- | -------------------------- | ----------- | ------------------ |
| Context 이해 | AI가 어떤 파일을 참고했는지 설명할 수 있음  | 대략 이해함      | 파일 접근 개념을 이해하지 못함  |
| Prompt 작성  | 목적, 입력, 출력, 제한 조건을 명확히 작성함 | 일부 조건이 빠짐   | 단순 명령만 입력함         |
| 결과물 검토     | 오류, 누락, 과장을 찾아냄            | 일부만 확인함     | AI 결과를 그대로 신뢰함     |
| 권한 이해      | 읽기/쓰기/승인 차이를 설명함           | 대략 이해함      | 권한 통제 필요성을 이해하지 못함 |

# 확장 실습 아이디어

## 1. 이메일 답변 초안 만들기

고객 문의 보고서를 바탕으로 고객별 1차 답변 이메일 초안을 작성한다. 단, 실제 발송은 하지 않는다.

## 2. 회의 준비 자료 만들기

고객 문의와 비용 리포트를 바탕으로 팀 회의용 1페이지 요약 자료를 만든다.

## 3. 자동화 개념 체험

매주 월요일 오전에 고객 문의 현황 요약 보고서를 만드는 자동화 프롬프트를 설계한다. 실제 예약 실행보다는 자동화 설계 개념 이해에 집중한다.

## 4. MCP 개념 소개

외부 도구와 AI Coworker를 연결하는 개념으로 MCP를 소개한다. 초급자에게는 “AI가 사용할 수 있는 도구를 표준 방식으로 연결하는 플러그”라고 설명한다.

# 실습 마무리 메시지

오늘 실습의 핵심은 AI가 스스로 모든 일을 완벽하게 처리한다는 것이 아니다. 핵심은 사람이 업무 목표와 경계를 정하고, AI가 파일을 읽고 초안을 만들며, 사람이 결과를 검토하고 승인하는 협업 구조를 이해하는 것이다.

Desktop AI Coworker는 “답변하는 AI”에서 “업무를 함께 수행하는 AI”로 넘어가는 중요한 전환점이다.

핵심은 초급자에게 **“AI가 내 컴퓨터에서 일을 할 수 있다”**보다 **“AI가 할 수 있는 일의 범위를 사람이 정하고, 결과를 사람이 검토한다”**를 먼저 이해시키는 것입니다. 그래서 실습은 자동화보다 **권한 부여 → 계획 확인 → 결과물 생성 → 검토 → 승인** 순서로 설계하는 것이 안전합니다.
