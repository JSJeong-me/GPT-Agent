아래는 위 설계 내용을 기반으로 정리한 **초급자 대상 Desktop AI Coworker 실습과정**입니다.

# Desktop AI Coworker 실습과정

## 실습 주제

**AI에게 업무를 맡기기: 파일 읽기, 보고서 작성, 표 정리, 승인 기반 실행 체험**

## 실습 목표

학습자는 실습을 통해 다음을 이해합니다.

1. 일반 챗봇과 Desktop AI Coworker의 차이
2. AI가 로컬 파일을 읽고 업무 맥락을 파악하는 방식
3. AI에게 업무를 지시할 때 필요한 프롬프트 구성
4. 결과물을 사람이 검토해야 하는 이유
5. 파일 변경, 저장, 이동 전 승인 절차의 중요성

---

# 전체 실습 흐름

| 단계  | 실습 내용                      | 핵심 개념               | 예상 시간 |
| --- | -------------------------- | ------------------- | ----: |
| 1단계 | 챗봇과 Desktop AI Coworker 비교 | 답변형 AI vs 업무 수행형 AI |   10분 |
| 2단계 | 실습 폴더 준비 및 권한 설정           | Context, Permission |   10분 |

| 4단계 | 고객 문의 분류 보고서 작성            | 문서 읽기, 분류, 보고서 생성   |   20분 |
| 5단계 | 영수증 데이터를 비용 리포트로 정리        | CSV 분석, 표 생성        |   20분 |
| 6단계 | 승인 기반 파일 정리 실행             | Human Approval      |   10분 |

# 1단계. 챗봇과 Desktop AI Coworker 비교

## 목적

일반 챗봇과 Desktop AI Coworker의 차이를 이해합니다.

## 진행 방법

먼저 일반 챗봇에게 다음과 같이 질문합니다.

```text
고객 문의 5건을 중요도별로 분류하고 대응 방안을 정리해 주세요.
```

이때 챗봇은 실제 고객 문의 파일을 볼 수 없기 때문에 정확한 답변을 할 수 없습니다.

그다음 Desktop AI Coworker에 실습 폴더를 연결하고 같은 요청을 합니다.

## 학습 포인트

일반 챗봇은 사용자가 입력한 텍스트 안에서 답변합니다.

Desktop AI Coworker는 사용자가 허용한 폴더와 파일을 읽고, 실제 업무 맥락을 바탕으로 결과물을 생성할 수 있습니다.

---

# 2단계. 실습 폴더 준비 및 권한 설정

## 목적

AI가 참고할 수 있는 업무 공간을 만들고, 접근 권한의 의미를 이해합니다.

## 실습 폴더 구조

```text
desktop-ai-coworker-practice/
├── 01_customer_inquiries/
│   ├── inquiry_001.txt
│   ├── inquiry_002.txt
│   ├── inquiry_003.txt
│   ├── inquiry_004.txt
│   └── inquiry_005.txt
├── 02_receipts/
│   └── receipts.csv
├── 03_product_docs/
│   ├── product_faq.md
│   └── service_policy.md
└── output/
```

## 진행 방법

Desktop AI Coworker에 위 실습 폴더를 연결합니다.

처음에는 **읽기 전용 권한**으로 시작합니다.

## 학습 포인트

AI에게 모든 파일과 폴더를 열어 주면 안 됩니다.

실습용 폴더처럼 제한된 공간만 허용해야 합니다.

읽기 권한과 쓰기 권한을 구분하는 것이 중요합니다.

---

# 3단계. AI에게 폴더 구조 파악시키기

## 목적

AI가 파일을 읽고 업무 맥락을 이해하는 과정을 체험합니다.

## 실습 프롬프트

```text
이 실습 폴더의 구조를 파악해 주세요.
각 폴더에 어떤 종류의 파일이 있는지 설명하고,
이 파일들을 바탕으로 어떤 업무를 수행할 수 있을지 5가지 아이디어를 제안해 주세요.

아직 파일을 수정하거나 새 파일을 만들지는 마세요.
먼저 계획만 보여 주세요.
```

## 기대 결과

AI는 다음 내용을 설명해야 합니다.

| 폴더                    | 의미                 |
| --------------------- | ------------------ |
| 01_customer_inquiries | 고객 문의 파일           |
| 02_receipts           | 비용 또는 영수증 데이터      |
| 03_product_docs       | 제품 FAQ 및 서비스 정책 문서 |
| output                | 결과물 저장 폴더          |

AI는 가능한 업무 예시도 제안합니다.

예를 들면 다음과 같습니다.

1. 고객 문의 유형 분류
2. 고객 문의 긴급도 판단
3. 고객별 1차 답변 초안 작성
4. 월간 비용 리포트 생성
5. FAQ 기반 응답 자료 작성

## 학습 포인트

AI에게 바로 작업을 시키기 전에 먼저 **업무 계획을 요청**해야 합니다.

이 단계는 AI가 어떤 파일을 읽고 어떤 작업을 하려는지 확인하는 과정입니다.

---

# 4단계. 고객 문의 분류 보고서 작성

## 목적

AI가 여러 개의 고객 문의 파일을 읽고, 구조화된 보고서를 만드는 과정을 체험합니다.

## 실습 프롬프트

```text
01_customer_inquiries 폴더의 고객 문의 5건을 읽고,
03_product_docs 폴더의 product_faq.md와 service_policy.md를 참고하여
고객 문의 분류 보고서를 작성해 주세요.

보고서에는 다음 항목을 포함해 주세요.

1. 문의 번호
2. 고객명
3. 문의 유형
4. 긴급도
5. 예상 원인 또는 핵심 이슈
6. 권장 대응 방법
7. 담당 부서
8. 1차 답변 초안

결과는 markdown 형식으로 작성하고,
파일명은 output/customer_inquiry_report.md 로 저장해 주세요.

파일을 저장하기 전에 먼저 어떤 파일을 읽고 어떤 파일을 생성할지 계획을 보여 주세요.
```

## 기대 산출물

```text
output/customer_inquiry_report.md
```

## 결과물 검토 항목

학습자는 생성된 보고서를 확인합니다.

| 검토 항목    | 확인 내용                        |
| -------- | ---------------------------- |
| 문의 내용 반영 | 고객 문의가 정확히 요약되었는가?           |
| 유형 분류    | 오류, 배송, 사용법, 불만 등이 잘 구분되었는가? |
| 긴급도 판단   | 긴급도 판단이 타당한가?                |
| 대응 방법    | 고객 상황에 맞는 대응 방안인가?           |
| 답변 초안    | 실제 고객에게 보내도 무리가 없는 표현인가?     |
| 환각 여부    | 없는 정보를 만들어 내지 않았는가?          |

## 학습 포인트

AI 결과물은 최종본이 아니라 **초안**입니다.

특히 고객 대응, 계약, 비용, 법적 책임과 관련된 내용은 사람이 반드시 검토해야 합니다.

---

# 5단계. 영수증 데이터를 비용 리포트로 정리

## 목적

AI가 CSV 데이터를 읽고 비용 리포트를 만드는 과정을 체험합니다.

## 실습 프롬프트

```text
02_receipts/receipts.csv 파일을 읽고 월간 비용 리포트를 작성해 주세요.

다음 결과물을 만들어 주세요.

1. 카테고리별 합계
2. 전체 비용 합계
3. 가장 큰 비용 항목 Top 3
4. 비용 사용 패턴에 대한 짧은 설명
5. 다음 달 비용 관리 제안

가능하면 스프레드시트 파일로 저장하고,
파일명은 output/monthly_expense_report.xlsx 로 해 주세요.

저장하기 전에 먼저 생성 계획을 보여 주세요.
```

## 기대 산출물

```text
output/monthly_expense_report.xlsx
```

환경에 따라 다음 파일로 생성될 수도 있습니다.

```text
output/monthly_expense_report.csv
output/monthly_expense_report.md
```

## 결과물 검토 항목

| 검토 항목    | 확인 내용                |
| -------- | -------------------- |
| 전체 합계    | 총 비용 계산이 맞는가?        |
| 카테고리별 합계 | 분류별 금액이 정확한가?        |
| Top 3 비용 | 큰 비용 항목이 제대로 정렬되었는가? |
| 파일 저장 위치 | output 폴더에 저장되었는가?   |
| 원본 보존    | 원본 CSV가 변경되지 않았는가?   |

## 학습 포인트

AI는 표 생성과 계산을 도와줄 수 있지만, 숫자는 반드시 사람이 검산해야 합니다.

---

# 6단계. 승인 기반 파일 정리 실행

## 목적

AI가 파일을 변경하거나 이동하기 전에 사람이 계획을 확인하고 승인하는 절차를 체험합니다.

## 1차 프롬프트: 계획 요청

```text
현재 output 폴더에 생성된 결과물을 기준으로
최종 제출용 폴더 구조를 제안해 주세요.

단, 실제 파일 이동이나 이름 변경은 아직 하지 마세요.
먼저 다음 형식으로 계획을 보여 주세요.

- 변경할 파일
- 새 파일명
- 이동할 위치
- 변경 이유
- 위험도
- 사용자의 승인 필요 여부
```

AI가 파일 정리 계획을 제안하면 학습자는 내용을 검토합니다.

## 2차 프롬프트: 승인 후 실행

```text
제안한 계획 중 위험도가 낮은 항목만 실행해 주세요.
원본 파일은 삭제하지 말고, 복사본만 만들어 주세요.
```

## 기대 결과

AI는 원본 파일을 삭제하지 않고, 결과물 복사본만 생성합니다.

## 학습 포인트

중요한 작업은 바로 실행하지 않습니다.

먼저 계획을 보고, 사람이 검토한 뒤 승인해야 합니다.

특히 다음 작업은 반드시 승인 절차가 필요합니다.

| 작업          | 승인 필요 이유     |
| ----------- | ------------ |
| 파일 삭제       | 복구가 어려울 수 있음 |
| 파일 덮어쓰기     | 원본 손상 가능     |
| 외부 전송       | 정보 유출 가능     |
| 이메일 발송      | 잘못된 발송 위험    |
| 계약·비용 관련 처리 | 책임 문제 발생 가능  |

# 7단계. 회고 및 업무 적용 토론

## 목적

학습자가 Desktop AI Coworker를 자신의 실제 업무에 어떻게 적용할 수 있을지 생각해 봅니다.

## 개인 회고 질문

```text
1. 오늘 실습에서 AI가 가장 잘한 일은 무엇인가요?
2. AI 결과물 중 사람이 반드시 검토해야 할 부분은 무엇인가요?
3. 내 업무에서 AI Coworker에게 맡길 수 있는 반복 업무는 무엇인가요?
4. AI에게 맡기면 안 되는 업무는 무엇인가요?
5. 더 좋은 결과를 얻기 위해 프롬프트에 추가해야 할 조건은 무엇인가요?
```

## 팀 토론 주제

```text
Desktop AI Coworker는 단순 자동화 도구인가, 아니면 협업 동료인가?
```

## 토론 관점

| 관점                  | 설명                                    |
| ------------------- | ------------------------------------- |
| 단순 자동화 도구           | 정해진 규칙을 반복 실행                         |
| Desktop AI Coworker | 파일을 읽고, 계획을 세우고, 결과물을 만들며, 사람의 검토를 받음 |

```text
desktop-ai-coworker-practice/
└── output/
    ├── customer_inquiry_report.md
```

환경에 따라 스프레드시트 대신 markdown 또는 CSV 파일이 생성될 수 있습니다.

# 실습 핵심 요약

Desktop AI Coworker 실습의 핵심 흐름은 다음과 같습니다.

```text
폴더 연결
→ 파일 구조 파악
→ 작업 계획 요청
→ 보고서/표 생성
→ 사람이 결과 검토
→ 승인 후 파일 정리
→ 실제 업무 적용 아이디어 도출
```

초급자에게 가장 중요한 메시지는 다음입니다.

**Desktop AI Coworker는 모든 일을 자동으로 처리하는 도구가 아니라, 사람이 정한 범위 안에서 업무를 도와주는 AI 동료입니다.**

따라서 실습의 핵심은 자동화 자체가 아니라 다음 세 가지입니다.

1. **AI에게 어떤 파일을 보여 줄 것인가**
2. **AI에게 어떤 작업을 맡길 것인가**
3. **AI가 만든 결과를 사람이 어떻게 검토하고 승인할 것인가**
